# Skills from A Smart Bear

A GitHub Sites website generated by [Astro](https://astro.build) (using the
[Starlight](https://starlight.astro.build) docs theme) that houses Claude Code
skills implementing frameworks and algorithms invented by Jason Cohen over two
decades of articles at [A Smart Bear](https://longform.asmartbear.com) and the
new book *[Hidden Multipliers](https://hiddenmultipliers.com)*.

The site is deployed via GitHub Actions to **GitHub Pages** at
**skills.asmartbear.com**. Runtime / package manager is **Bun**. The git
remote is **`origin`** (github.com/asmartbear/asb-skills); push to `main`
triggers deploy.

## Two classes of skill

`.claude/skills/` holds both: **public `asb-*` skills** (have a wrapper at
`src/content/skills/<name>.mdx`; published on the website; must be fully
self-contained and Claude-Code-agnostic) and **dev-only skills** (no `asb-`
prefix, no wrapper; unrestricted internal tooling, never distributed). The
authoritative rules are in `.claude/skills/CLAUDE.md` — **read it before
touching any skill.**

## Kinds of changes

| Change | Files to touch | Verify with |
| :----- | :------------- | :---------- |
| New public skill | `/create-asb-skill` workflow → `.claude/skills/asb-<n>/SKILL.md` + `src/content/skills/asb-<n>.mdx` | `bun run lint && bun run build`; `/exercise-asb-skill asb-<n>` |
| Edit a published skill | the same two files, kept consistent | `bun run lint`; re-run `/exercise-asb-skill asb-<n>` |
| Dev-only skill / tooling | `.claude/skills/<n>/` (no `asb-` prefix, no wrapper) | `bun run lint` |
| Site change (pages, components, styles) | `src/` (see `src/CLAUDE.md`) | `bun run check && bun run build`; eyeball via `bun run dev` |
| Skill-idea backlog | `BACKLOG.md` only | nothing |
| Scripts / lint rules | `scripts/*.ts` (see `scripts/CLAUDE.md`) | `bun run lint` |
| Publish to production | push to `main` | `bun run deploy-and-wait` |

## How skill content is loaded

There is **one source of truth per skill**: the `SKILL.md` file at
`.claude/skills/<name>/SKILL.md`. There is no sync step and no generated copy.

The `src/content/skills/<name>.mdx` file (for public skills) holds page-level
metadata (title, summary, ordering) plus optional marketing wrapper content.
At build time, Astro reads both files and joins them to render the per-skill
page. The presence/absence of the `.mdx` is itself the index of what's public.

## External source corpus (for skill *development* only)

Two large bodies of Jason's writing live outside this repo and are **never
committed and never deployed**. Read them when you need to understand the
framework a new skill should encode:

- **A Smart Bear articles**: `/Users/jcohen/Obsidian/Longform/Projects/Articles/Content/*.md`
- **Hidden Multipliers book**: `/Users/jcohen/Obsidian/Longform/Projects/Hidden Multipliers/` — start by reading its own `CLAUDE.md`, which maps `Overview.md`, `Thesis.md`, `Chapters.md`, `Audience.md`, `Marketing.md`, `Prose Style.md`, `Prose Samples.md`, and chapter bodies under `Content/`.

When building a new `asb-*` skill, use these as research material to extract
the framework, then **re-state it from scratch** in the SKILL.md. Do not copy
prose verbatim, do not import, do not link to those files from skill content.
The published skill must explain itself.

## Commands

```
bun install              # install deps
bun run dev              # Astro dev server (http://localhost:4321)
bun run build            # static build → dist/
bun run check            # astro check (type + content schema)
bun run lint             # lint all skills + wrappers
bun run link <name>      # symlink skill into ~/.claude/skills (cross-repo use; see scripts/CLAUDE.md)
bun run unlink <name>    # remove that symlink
bun run deploy-and-wait  # git push, then watch the GitHub Pages deploy (requires `gh`)
```
